1. Digital Encoding 数字编码 (p16–17)

用数字信号 (digital signal) 传送数字数据 (digital data)，需要把比特序列映射为电平变化。常见编码方案：

编码 Scheme规则 Rule优点 Pros缺点 Cons
NRZ-L
Non-Return-to-Zero Level		 0 = 高电平, 1 = 低电平（或反之） 简单，带宽低 长串相同比特无跳变 → 无法同步；有直流分量
NRZ-I
NRZ-Invert		 遇 1 跳变，遇 0 保持 差分编码，抗极性反转 长串 0 仍无跳变
Manchester
曼彻斯特		 每位中间必有跳变：
0 = 高→低；1 = 低→高（IEEE 802.3 标准）		 自带时钟，无直流分量 带宽是 NRZ 的 2 倍
Diff Manchester
差分曼彻斯特		 位中间必跳变（用作时钟）；位开始处：0 跳变, 1 不跳变 抗极性反转 + 自带时钟 电路稍复杂
AMI
Bipolar (双极)		 0 = 0V；1 交替用 +V / −V 无直流分量；连 1 可同步 连 0 时无跳变

必记：Manchester 编码每位中间一定有跳变 → 自同步（self-clocking），所以以太网选它。

💡 为什么需要"自同步"编码？中文

接收端不知道发送端的时钟频率精确值，如果信号长时间不跳变（比如长串的 0），接收端就会"失同步"，无法判断每一位的边界。Manchester 编码强制每个比特中间有一次跳变，让接收端可以从信号本身恢复时钟，避免失同步问题。

代价是带宽翻倍——同样发送 1 bit 数据，Manchester 在线路上的信号变化次数是 NRZ 的 2 倍。

2. 4B/5B Conversion 4B/5B 编码 (p18)

把每 4 个数据位 映射为 5 个传输位，保证传输码中至少有 2 个 1（不会长时间无跳变），常用于 100Base-T 等。映射表如下：

Data (4B)Code (5B)Data (4B)Code (5B)
000011110100010010
000101001100110011
001010100101010110
001110101101110111
010001010110011010
010101011110111011
011001110111011100
011101111111111101

16 个 4-bit 模式只用了 2⁵=32 个 5-bit 模式中的 16 个；其余用作控制符（IDLE、Start/End delimiter 等）。

💡 为什么不直接用 NRZ-I 而要套 4B/5B？中文

NRZ-I 遇 1 跳变、遇 0 保持。如果原始数据出现 00000000，NRZ-I 信号会保持不变 → 失同步。

4B/5B 编码后，每 5 个码位至少包含 2 个 1，最大连续 0 个数被限制（不超过 3），保证再用 NRZ-I 发送时也能维持同步。

组合：原始数据 → 4B/5B 编码 → NRZ-I → 介质，是 100Mbps Fast Ethernet 的物理层方案。

3. AM / FM / PM 模拟调制 (p19–21)

用模拟信号 (analog signal) 传送数据时，要把信息"装载"到载波 (carrier wave) 上。载波 c(t) = A·cos(2π·f·t + φ) 有三个可改变的参数：振幅 A、频率 f、相位 φ。

AM — Amplitude Modulation 振幅调制

改变载波的振幅 A，频率和相位不变。

s(t) = [A + m(t)] · cos(2π·fc·t)

应用：调幅广播（AM 收音机）

FM — Frequency Modulation 频率调制

改变载波的频率 f，振幅和相位不变。

s(t) = A · cos[2π·(fc + k·m(t))·t]

应用：调频广播（FM 收音机），抗干扰强

PM — Phase Modulation 相位调制

改变载波的相位 φ，振幅和频率不变。

s(t) = A · cos[2π·fc·t + k·m(t)]

应用：数字调制 PSK 的基础

对比小结 Summary

项 AspectAMFMPM
抗噪声差好好
带宽需求窄宽宽
电路复杂度简单较复杂复杂

记忆口诀：振幅—频率—相位，分别对应 A — f — φ。数字版分别叫 ASK / FSK / PSK。

4. Nyquist Theorem 奈奎斯特定理 (p23–24)

无噪声信道的最大数据率 (channel capacity)：

Nyquist (Noiseless) C = 2 · B · log2(V)   (bit/s)
  • C = 信道容量 channel capacity (bps)
  • B = 带宽 bandwidth (Hz)
  • V = 信号电平数 number of signal levels（即每符号承载 log₂V 个 bit）

关键直觉：带宽越宽 → 单位时间可发的符号越多；电平数越多 → 每个符号能编码的 bit 越多。

例题 Example 1：B = 3 kHz, V = 8，求最大数据率
Step 1. 列公式 C = 2 · B · log₂(V)
Step 2. 代入：C = 2 × 3000 × log₂(8) = 2 × 3000 × 3
Step 3. C = 18 000 bps = 18 kbps
例题 Example 2：电话线 B=4kHz, 4 电平，求最大数据率
Solution: C = 2 × 4000 × log₂(4) = 8000 × 2 = 16 000 bps

5. Shannon's Theorem & SNR 香农定理与信噪比 (p25)

有噪声信道的最大数据率：

Shannon (Noisy) C = B · log2(1 + S/N)   (bit/s)
  • S/N = 信号功率 / 噪声功率（线性比，不是 dB）
  • SNRdB = 10 · log10(S/N)，所以反向 S/N = 10SNRdB/10
SNRdB = 10 · log10(S/N)   ⇔   S/N = 10SNRdB/10

常考陷阱：题目给的是 SNR 的 dB 值时，必须先换算回线性比 S/N 再代入！

例题 Example 1：B = 4 kHz, SNR = 30 dB，求最大数据率
Step 1. 把 dB 换算：S/N = 10^(30/10) = 10³ = 1000
Step 2. 代入：C = 4000 × log₂(1 + 1000) ≈ 4000 × log₂(1001)
Step 3. log₂(1024) = 10，所以 log₂(1001) ≈ 9.97
Step 4. C ≈ 4000 × 9.97 ≈ 39 880 bps ≈ 40 kbps
💡 Nyquist 与 Shannon 怎么联用？中文

Nyquist 给出无噪声情况下的上限（和你选用几电平有关）。Shannon 给出有噪声情况下的绝对上限（和电平数无关，只和带宽 + SNR 有关）。

实际工程中，C_max = min(CNyquist, CShannon)。即两个上限同时成立，取较小者。

题目通常分开问；如果两个都给，先算两个再取最小。

Why Layering? 为什么分层？

  • Modularity 模块化：每层只关心自己的任务，对外暴露简单接口
  • Hide complexity 隐藏复杂度：应用层不必懂物理层细节
  • Easy to modify / replace 易于修改：换一种物理介质，上层无感知（如以太网换光纤）
  • Standardization 标准化：不同厂商的设备可在层接口上互操作
  • Easier debugging 易于调试：故障可定位到具体层
💡 数据如何在分层中"打包/解包"？中文

发送方从上到下封装 (encapsulation)：每层在数据前加自己的 header（链路层还可能加 trailer）。

接收方从下到上解封 (decapsulation)：每层剥掉自己的 header，把 payload 交给上一层。

这就是为什么以太网帧的 payload 里又能装一个 IP 包，IP 包里又能装一个 TCP 段——一个层一个信封。

7. Network Topologies 网络拓扑 (p62–71)

教师特别提示：可能考"什么情况下用什么拓扑"——务必记住每种拓扑的适用场景。

拓扑优点 Pros缺点 Cons适用场景 Use Case
Bus 总线 布线少、便宜、易扩展 主线一断全网瘫；冲突多 早期小型 LAN；现在很少用
Star 星形 中心控制易管理；单节点故障不影响他人 中心 hub/switch 是单点故障；电缆多 现代以太网 LAN（家庭/办公室主流）
Ring 环形 无冲突（令牌环）；公平访问 任一节点/链路故障影响整环；增/删节点麻烦 令牌环 (Token Ring)、FDDI；工业控制网络
Mesh 网状 高可靠性，多路径，无单点故障 布线极多 (N(N−1)/2)；成本高 骨干网 (backbone)、军事/关键系统
Tree 树形 层次清晰，便于分段管理 根节点故障影响子树 大型园区网、CATV 有线电视网

🎯 Self-Test 章末自测 (6 题)

Q1. 下列哪种编码自带时钟，每位中间必有跳变？

  • A. NRZ-L
  • B. NRZ-I
  • C. Manchester
  • D. AMI
答案 C. Manchester 编码每位中间有跳变（0=高→低、1=低→高），接收端可从信号本身恢复时钟，故称为自同步。NRZ 系列长串相同位时无跳变；AMI 长串 0 时无跳变。

Q2. 用 Nyquist 公式：带宽 B = 6 kHz，电平数 V = 8，求最大数据率（bps）。

C = bps

C = 2·B·log₂(V) = 2 × 6000 × log₂(8) = 2 × 6000 × 3 = 36 000 bps

Q3. 用 Shannon 公式：B = 2 kHz, SNR = 30 dB，求理论最大数据率（bps，取整数）。

C ≈ bps

先把 dB 换线性：S/N = 10^(30/10) = 1000。
C = B · log₂(1 + S/N) = 2000 × log₂(1001) ≈ 2000 × 9.97 ≈ 19 940 ≈ 20 000 bps

Q4. 在 4B/5B 编码中，4-bit 数据 0000 被映射为：

  • A. 00000
  • B. 11110
  • C. 01010
  • D. 10101
答案 B. 0000 → 11110（见映射表）。注意 4B/5B 不允许码字本身有过多连 0，所以 0000 这种全 0 数据必须映射成含若干 1 的码字。

Q5. 一家公司要在多栋楼之间组建可靠网络，对容错要求极高（任一链路失效都不能整网失联），最适合的拓扑是？

  • A. Bus
  • B. Star
  • C. Mesh
  • D. Ring
答案 C. Mesh. 网状拓扑提供多条冗余路径，任意一条链路或节点失效都不影响其它节点通信。Star 受限于中心 hub 的单点故障；Bus 主线一断全瘫；Ring 任一节点故障影响整环（除非用双环）。

Q6. 简答：列举至少 3 个"为什么要分层"的理由。

参考答案（任答 3 个即可）：
  1. Modularity 模块化：每层职责单一，便于设计与维护
  2. Hides complexity 隐藏复杂度：上层不必关心下层实现细节
  3. Easy to replace 易替换：换一种物理介质或链路协议，上层无需修改
  4. Standardization 标准化：层间接口统一，不同厂商设备可互操作
  5. Easier debugging 易调试：故障可定位到具体层
