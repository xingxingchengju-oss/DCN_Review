以太网 CSMA/CD、二进制指数退避、WLAN CSMA/CA

1. 为什么要最小帧长？ Minimum Frame Size (p13)

核心原因：CSMA/CD 必须保证发送方在帧发完之前就能检测到碰撞，否则它发完就走，根本不知道发生了冲突，也不会重传。

场景分析

  1. A 开始发送，信号沿介质传播，需要时间 τ（单程传播延迟）才能到达最远点 B
  2. 假设 B 在 A 发完之前的最后一刻才开始发送 → 双方信号在介质上碰撞
  3. 碰撞信号要再传 τ 回到 A → A 才知道"我冲突了"
  4. 所以从 A 开始发到 A 检测到碰撞，最坏情况要 2τ 时间
  5. 要让 A 在帧发完前还能检测到这个碰撞 → 帧的传输时间 Tframe ≥ 2τ

2. 最小帧长公式 (p17)

基本不等式 Tframe ≥ 2τ = 2L / v
最小帧长（比特） Lmin = R · 2τ = R · 2L / v
  • Tframe = 帧的传输时间（s）
  • τ = 单程传播延迟（s）
  • L = 网络长度 / 端到端距离（m）
  • v = 信号传播速度（m/s，铜缆约 2×10⁸，光纤约 2×10⁸）
  • R = 数据率（bps）
  • Lmin = 最小帧长（bit）
例题 1：10 Mbps 以太网，最长 2.5 km，信号速度 2×10⁸ m/s，求最小帧长
Step 1. 单程传播延迟 τ = L / v = 2500 / (2×10⁸) = 1.25×10⁻⁵ s = 12.5 μs
Step 2. 往返延迟 2τ = 25 μs
Step 3. 最小帧长（比特）= R · 2τ = 10×10⁶ × 25×10⁻⁶ = 250 bit
Step 4. 换算字节：250 / 8 = 31.25 byte → 实际以太网取 64 byte = 512 bit（留余量，含 18 字节头 + 46 字节最少数据）
例题 2：要把以太网做到 100 Mbps，距离要变成多少？
分析： R 增大 10 倍，要保持 Lmin=512 bit，则 2τ 缩小 10 倍 → 距离 L 缩小 10 倍 → 250 m
结论： 100Base-T 的网段长度被限制在约 100 m（实际还要更短）。这就是更高速以太网必须降级网段长度的物理原因。
💡 为什么以太网设最小帧 64 字节而不是 31 字节？中文

上面算出"理论"最小 250 bit ≈ 32 字节，但 IEEE 802.3 规定 64 字节 = 512 bit。多出来的余量是为：

  • 中继器/集线器的处理延迟（每个都让 2τ 增大）
  • 同轴电缆比纯铜慢一点
  • 给标准留弹性，便于未来扩展

所以记住：以太网最小帧 = 64 字节 = 512 bit。

3. Binary Exponential Back-off 二进制指数退避 (p35)

在 CSMA/CD 中，当检测到冲突后，发送方不立刻重传，而是等一个随机时间后再试，避免同步性重碰撞。

算法步骤 Algorithm

  1. 第 n 次冲突后，让 k = min(n, 10)
  2. 从集合 { 0, 1, 2, …, 2k − 1 } 中均匀随机选一个数 r
  3. 等待 r 个时间槽 (slot time) 后重传
    （时间槽 = 2τ，以太网中为 51.2 μs）
  4. 若第 16 次还失败，放弃，上报错误

每次冲突后的等待区间

例题：连续发生 3 次冲突后，最长可能要等多少 μs？（slot = 51.2 μs）
Step 1. n=3, k=3, 等待集合 {0,1,...,7}，最大 r=7
Step 2. 最长 = 7 × 51.2 μs = 358.4 μs
💡 为什么用"二进制指数"？中文

设想 N 个站同时想发：

  • 第一次冲突，每个站从 {0,1} 随机选 → 仍有 50% 概率再次冲撞
  • 下一次区间扩大到 {0,1,2,3}，冲突概率降到 25%
  • 区间指数级扩大 → 冲突概率指数级下降

这种自适应让网络在低负载时延迟低（区间小，几乎立刻重传），在高负载时也不爆炸（区间扩大避免持续冲突），是个非常优雅的退避机制。

4. WLAN & CSMA/CA 无线局域网 (p50–51)

为什么无线不能用 CSMA/CD？

无线网卡无法边发边听（半双工，发射时本机天线信号远强于其它信号），所以检测不到冲突，CSMA/CD 失效。改用 CSMA/CA：避免冲突而不是检测。

Hidden Terminal 隐藏终端问题

A 和 C 都能与 AP 通信，但相互听不到对方。A 听到信道空闲就发，同时 C 也以为空闲也发 → 两个信号在 AP 处同时到达，相互干扰。这就是隐藏终端 (hidden terminal)。

CSMA/CA 基本流程

  1. 站点想发送 → 侦听信道
  2. 若空闲，等待一个 DIFS（分布式协调间隔）
  3. 再等一个随机退避时间（避免多个站同时抢发）
  4. 发送数据帧
  5. 接收方收到正确帧后等一个 SIFS（短间隔，比 DIFS 短），回 ACK
  6. 发送方没收到 ACK → 假定冲突或丢包，按二进制指数退避重传

RTS / CTS 机制 — 解决隐藏终端

  1. A 想发送大数据 → 先发RTS (Request To Send) 给 AP（小包）
  2. AP 回CTS (Clear To Send)，所有能听到 AP的站（包括 C）都收到，知道"信道接下来要被占用 N 微秒"
  3. C 听到 CTS → 自觉沉默
  4. A 安全地发送大数据 → AP 收到回 ACK
💡 NAV 网络分配向量是什么？中文

RTS 和 CTS 包里都含有"接下来要占用信道多久"的字段。听到的站把这个时长写进自己的 NAV (Network Allocation Vector) 计时器，期间即使本地侦听信道空闲，也不发。这叫"虚拟载波侦听"，配合物理载波侦听一起判断信道是否可用。

🎯 Self-Test 章末自测 (6 题)

Q1. 100 Mbps 以太网，距离 500 m，v = 2×10⁸ m/s，求最小帧长（bit）。

Lmin = bit

τ = 500 / 2×10⁸ = 2.5 μs；2τ = 5 μs
Lmin = R · 2τ = 100×10⁶ × 5×10⁻⁶ = 500 bit

Q2. 为什么 CSMA/CD 需要规定最小帧长？

  • A. 节省带宽
  • B. 让接收方有足够缓冲
  • C. 保证发送方在帧发完之前能检测到冲突
  • D. 与 IP 头部对齐
C. 若帧太短，发送方发完后才"听到"远端的冲突回波，已经无法重传该帧。

Q3. 二进制指数退避第 4 次冲突后，随机槽数从哪个集合中选？

  • A. {0,1,2,3}
  • B. {0,1,...,7}
  • C. {0,1,...,15}
  • D. {0,1,...,31}
C. 第 n 次冲突，k=min(n,10)，区间是 {0,...,2k-1}。n=4 → 2⁴=16 → {0,1,...,15}。

Q4. 关于无线 LAN 不能直接用 CSMA/CD，主要原因是？

  • A. 无线带宽太低
  • B. 无线网卡发送时本机天线信号太强，无法同时听到远端微弱信号
  • C. 无线不支持以太网帧
  • D. 无线没有 MAC 地址
B. 无线收发同频段，发的时候听不到别人。所以改用 CSMA/CA 主动避免冲突，并用 ACK 确认。

Q5. A 和 C 都能与 AP 通信但听不到对方，A 在发送时 C 也开始发，导致 AP 收到的信号互相干扰。这种情况叫：

  • A. Hidden Terminal 隐藏终端
  • B. Exposed Terminal 暴露终端
  • C. Capture Effect 捕获效应
  • D. Near-Far Problem 远近问题
A. 隐藏终端：两站对 AP 都可见但相互"看不见"。RTS/CTS 机制就是为解决这个问题设计的。

Q6. CSMA/CD 与 CSMA/CA 的核心区别是？

  • A. CSMA/CD 用 ACK，CSMA/CA 不用
  • B. CSMA/CD 用于无线，CSMA/CA 用于有线
  • C. 二者完全相同
  • D. CSMA/CD 检测冲突；CSMA/CA 避免冲突
D. CD = Collision Detection（事后检测），CA = Collision Avoidance（事前避免）。无线无法边发边听，所以只能"避免"。同时 CSMA/CA 需要 ACK 来间接判断冲突，CSMA/CD 不需要。
